# 03-Knowledge

Atomic knowledge notes, organized by topic/area, not by source. One idea per note where
practical; cross-link liberally instead of duplicating content across notes.

If the daily pipeline distills recurring heuristics, lessons or decision patterns, they
land under a subfolder here once `config.taxonomy.heuristics` points at one (unset by
default in MODE B; the owner can set it later without touching any code).

Evergreen notes in this folder do not carry a `status` field (absence means `active`);
only dynamic notes (things AI writes or regularly updates) need one.
