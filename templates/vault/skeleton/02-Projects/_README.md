# 02-Projects

One subfolder per active project. Each project subfolder should carry a `_STATUS.md`
(or similarly named status dashboard) with a `status` field (see the trust-axis table in
`CLAUDE.md`) so the aging audit and the retrieval hooks can reason about how current it
is.

Archive or clearly mark inactive projects rather than deleting them (supersede, never
delete, is the vault-wide rule).
