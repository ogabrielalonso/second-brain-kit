# 01-MOCs

Maps of Content: topic-first entry points into the vault. Each MOC links out to the
atomic notes under `03-Knowledge/` and `02-Projects/` that belong to one topic, so a
reader (human or AI) can start from a subject and fan out, instead of browsing folders.

One MOC per topic the owner actually returns to, not one per source. If `config.taxonomy.
moc_map` is set, deterministic appliers file new notes into the mapped MOC's "Recent
entries (auto)" section automatically; weekly curation (cheap model) tidies the rest by
hand.

Do not put raw source material here, only links and short framing text.
